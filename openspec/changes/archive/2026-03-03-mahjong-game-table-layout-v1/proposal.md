## Why

目前專案只有牌面渲染的基礎建設，尚未定義遊戲的整體畫面結構。需要先確立第一版桌面布局，作為後續 UI 實作的基準。

## What Changes

- 定義四列式桌面布局：每一家（東南西北）各佔一列
- 定義各列顯示的資訊：手牌（AI 為背面）、副露、最新棄牌
- 定義最新棄牌的顯示邏輯：顯示於各列最右側，吃碰槓後更新
- 定義鍵盤驅動的互動模式：吃/碰/槓/胡等動作全部使用快捷鍵
- 定義棄牌區以 dialog 熱鍵開啟（不常駐顯示）
- 玩家列（南家）與狀態資訊列合併為同一列

## Capabilities

### New Capabilities

- `game-table-layout`: 四列式主桌面布局規格，包含各家資訊顯示規則與最新棄牌區邏輯
- `keyboard-interaction`: 鍵盤驅動互動模式規格，定義快捷鍵與 UI 只顯示狀態的設計原則

### Modified Capabilities

（無）

## Impact

- `tui/src/index.tsx`：目前為牌面預覽用途，後續需重構為正式桌面布局
- 未來 Zig 核心的遊戲狀態需對應到各列的顯示資料結構
