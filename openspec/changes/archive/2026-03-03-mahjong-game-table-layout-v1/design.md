## Context

目前 TUI 入口（`tui/src/index.tsx`）僅用於牌面渲染預覽。需要定義正式的桌面布局結構，作為遊戲主畫面的基礎。

終端機特性：
- 橫向空間充足，縱向空間有限
- 玩家手牌（13-16 張）橫向排列，寬度是主要限制
- 假設最低可用終端大小為 120×30，建議滿版使用

## Goals / Non-Goals

**Goals:**
- 定義四列式主桌面的結構與各區域職責
- 定義最新棄牌的狀態機（何時顯示、何時清除）
- 確立鍵盤驅動的設計原則
- 作為後續 UI 元件實作的規格基準

**Non-Goals:**
- 實際的鍵盤事件綁定實作
- 遊戲邏輯（抽牌、勝負判定）
- 棄牌 dialog 的詳細設計
- 多解析度響應式細節

## Decisions

### 決策 1：四列式布局，每家一列

每位玩家（東南西北）各佔一個水平列。玩家（南家）固定在最底列。

```
┌──────────────────────────────────────────────────────┐
│ 北  🀫×13  副露:[東東東]                    │  5萬 ← │
├──────────────────────────────────────────────────────┤
│ 西  🀫×11  副露:[南南南]                    │  3筒 ← │
├──────────────────────────────────────────────────────┤
│ 東  🀫×13  副露:—                           │       │
├──────────────────────────────────────────────────────┤
│ 你  1萬2萬3萬4萬5萬 1筒2筒 東東 5條6條7條  │  7條 ← │
│     a  s  d  f  g   h  i   j  k   l  ;  '           │
│ 東風三局  剩44張  c=吃 p=碰 k=槓 h=胡  r=棄牌       │
└──────────────────────────────────────────────────────┘
```

**理由**：終端機橫向空間充足，四列能清楚區隔四家，且符合「最低揭露」原則（AI 只顯示張數）。

### 決策 2：最新棄牌顯示於各列最右側

右側保留固定寬度欄位，常態為空，有新棄牌時顯示該張牌。

最新棄牌狀態機：
- **清空** → 某家打出牌 → **顯示該牌**
- **顯示該牌** → 被下一家摸走（無人搶） → **清空**
- **顯示該牌** → 有人吃/碰/槓 → 那人再打出新牌 → **更新為新牌**

任何時刻整個桌面最多只有一列的右側有棄牌顯示。

**理由**：「最新棄牌」是當下所有玩家需要關注的唯一牌，突顯它能幫助玩家快速決策。

### 決策 3：玩家列與狀態資訊合併

玩家（南家）那一列包含三個子列：
1. 手牌（正面顯示）
2. 按鍵提示（對應每張手牌的打牌快捷鍵）
3. 狀態列（局況 + 可用動作快捷鍵）

**理由**：狀態資訊只與玩家操作相關，放在玩家列最自然。

### 決策 4：UI 只顯示，互動全靠快捷鍵

- 打牌：按對應手牌位置的鍵（a/s/d/f/g...）
- 吃碰槓胡：單鍵快捷鍵（c/p/k/h）
- 棄牌區查看：熱鍵開啟 dialog
- 不使用滑鼠、不使用選單導航

**理由**：終端機遊戲的核心體驗，減少視線移動，保持專注。

## Risks / Trade-offs

- **終端寬度不足** → 手牌過多時（16張副露後）可能超寬。緩解：牌面字元寬度需精確計算，未來可考慮縮短牌面表示。
- **AI 列資訊過少** → 玩家可能感到沉悶。緩解：副露牌組正面顯示，提供足夠線索。
- **快捷鍵學習曲線** → 緩解：狀態列常態顯示可用快捷鍵。
